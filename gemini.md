# B.L.A.S.T. Project Constitution

## Framework Rules
1. **HTML**: Semantic HTML5. Flat DOM tree.
2. **CSS**: Tailwind CSS via CDN. Focus on "Modern Luxe" (whitespace, fade-ups, magnetic buttons).
3. **JS**: Vanilla JavaScript only for animations and form submissions. No external libraries unless explicitly approved.

## Behavioral Rules
1. **Self-contained code**: All code must be copy-paste ready for a GoHighLevel "Custom HTML" element (inline CSS/JS or internal `<style>`/`<script>` tags).
2. **Deterministic execution**: No code is written until the 5 Discovery Questions are answered and the Blueprint is approved.
3. **Mobile-first**: All code must be mobile-first and optimized for high-converting sales.

## Architectural Invariants
1. Separate structure from styling conceptually, but deliver as a single self-contained unified block for GHL.
2. Ensure pristine typography, exact spacing, and curated color palettes.
3. Adhere strictly to the B.L.A.S.T protocol (Blueprint -> Link -> Architect -> Stylize -> Trigger).
